Checks if there's a new Voltage Request.

It automatically checks if the CV setting is <= 3.6V. If the condition was met, it Disables the UVL Setting in order to prevent a Fault during low voltage operation

This also checks whether the New Voltage(CV) Value to Be written is Higher or lower than the previous Voltage(CV) and decides either to increment or decrement the voltage setting of the Inno Pro

bool InnoPro_Application::InnoPro_Process_Voltage ( bool  bVoltIncrease)

Handles all I2C Configurations to be written to Inno Pro.

This function Writes to all Inno Pro registers one at a time A certain delay is implemented on every write transaction to make sure that Inno Pro command rate for Write Transaction is not violated (10ms). A wait time delay is added after the configurations was written

Precondition
clock_TimeUpdate() function should have been running at 200us to generate ms
I2C_Write16() Driver should be already Configured and is able to send I2C commands
Parameters
bVoltIncrease- True or False Value to Increase or Decrease Voltage
Returns
True - after I2C commands are written and certain Time(~75ms) has elapsed

◆ InnoPro_Vbus_Switch_Control()

void InnoPro_Application::InnoPro_Vbus_Switch_Control ( float  fEnableVben)

Vbus Switch Control (VBEN Control)

Used to Enable or Disable Vbus Switch By Writing a value to VBEN Register

This function stores the old value it received. 
If the Input parameter is the same with the previous 
it will not execute the I2C Command
Note
When VBEN is Disabled, Watchdog is Enabled As Default so Watchdog Timer is needed to be Re-Disabled Thus this also writes to Watchdog Register
When a fault condition occurs, VBEN is automaticaly disabled,
to Activate VBEN use a value different from the previous stored value.
Since the input parameter of this function is usually set to a value of 1,
The user can use a value greater than 1 to re-enable VBEN.
Precondition
None
Parameters
fEnableVben- Value to Enable VBEN Input Values: 0: Disable , 1: Enable , >1 : Enable
Returns
None

◆ InnoPro_Write_VI()

void InnoPro_Application::InnoPro_Write_VI ( float  fSetPtCV,
float  fSetPtCC 
)

Output Voltage Control and Constant Current (CC) Control.

Used to update the Value of CV Register This is also used to compute for UVA and OVA settings as some percentage CV Also Used to update the Value of CC Register

This function stores the old values it received. 
If the Input parameters is the same with the previous 
it will not execute the I2C Command
Note
This function must run on Main loop. It Uses Elapsed Time function so this must not be blocked. Minimum of ~ 30ms is needed to complete the Increase/Decrease of Voltage. And Another ~ 10ms delay to complete the states of this function. It is recommended that User must only update VI parameters after 50ms.
Precondition
If the platform is not an Arduino clock_TimeUpdate() function should have been running at 200us to generate milli second
I2C_Write16() Driver should be already Configured and is able to send I2C commands Constant Current value depends on the Sense Resistor Value
Parameters
fSetPtCV- Output Voltage Set Point Value Input Range: 3 to 24 V (in 10mv Step)
fSetPtCC- Constant Current (CC) Set Point Value CC Range: 25% to 100% Input Range: up to 6A
Returns
None
